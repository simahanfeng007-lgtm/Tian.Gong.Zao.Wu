============================================================
一、L0 定义
============================================================

L0 是天工造物的生命数据原子层。
它只定义可序列化、可审计、可哈希、可回放、可恢复、可演化的最小事实语言。
它不执行任何真实动作。

硬规则：
- 只允许 Python 标准库。
- 所有 dataclass 必须 frozen=True, slots=True。
- 采用 Result-first。
- Canonical JSON 为第一事实格式。
- 不允许真实 IO、网络、模型调用、工具调用、插件加载、进程启动。

四、自动测试清单
============================================================

1. test_l0_no_outer_imports.py：L0 不 import L1-L6。
2. test_l0_no_third_party_imports.py：L0 不 import 第三方库。
3. test_l0_no_io.py：L0 不调用 open、Path.read_text、Path.write_text、socket、subprocess、requests 等真实 IO。
4. test_l0_dataclass_frozen.py：所有 L0 dataclass 必须 frozen=True, slots=True。
5. test_l0_serialization.py：所有 L0 对象可稳定转 primitive / canonical JSON。
6. test_l0_stable_hash.py：所有可事实化对象 stable_hash 可重复。
7. test_l0_no_system_pollution_words.py：L0 核心类名不得出现 ToolExecutor、PluginHost、MemorySystem、Frontend、Gateway、Database、Shell、HTTPClient 等上层实现词。
8. test_l0_ref_objects.py：所有 Ref 类只保存引用事实，不承载执行器、客户端、回调函数或真实资源句柄。
9. test_l0_enum_stability.py：枚举值发布后不得删除或重命名，只能新增或 deprecated。
10. test_l0_result_first.py：普通校验函数返回 CoreResult，不用异常作为常规控制流。

六、最终结论
============================================================

L0 的目标不是写复杂逻辑，而是建立整个天工造物生命体的共同事实语言。
L0 是数据原子层，不是大脑、肌肉、工具、插件或前端。
