## 本轮审计方式

新增 `scripts/desktop_human_visual_click_audit_l6706.py`，使用真实 Tk 窗口和本地桥接服务进行确定性审计：

- 启动本地桥接服务。
- 打开桌面端主窗口。
- 逐页点击：聊天、执行、任务、观测、文件、工作区、连接器、记忆、自我迭代、四路径、安装、设置、Hooks。
- 验证非聊天页滚动容器。
- 验证聊天发送、流式收口、普通聊天不触发待确认。
- 验证聊天刷新后滚动钉底。
- 验证文件上传、写入目录授权、连接器注册、自检、主题切换、Provider Key 保存与 auto/provider 切换。

## 关键修复

1. **Tk 线程安全**
   - 发现后台流式线程直接调用 Tk `after` 存在线程异常风险。
   - 已改为 `queue.Queue` + 主线程轮询执行 UI 更新。

2. **全页滚动可达**
   - 非聊天页面统一包入外层 Canvas + Scrollbar。
   - 解决低高度窗口下卡片下半部分不可点击的问题。

3. **聊天钉底**
   - 流式刷新和页面重建后强制 `see("end")` 与 `yview_moveto(1.0)`。

4. **主题体感**
   - 主题切换由局部 repaint 改为整壳重建。
   - 侧边栏、顶栏、状态栏、正文同步变色。

5. **启动默认模式**
   - 推荐启动脚本改为 `--backend-mode auto`。
   - 避免用户明明配置了 Key，却仍被启动脚本强行拉回 mock。

6. **点击回执**
   - 文件写入授权改目录选择。
   - MCP 注册、自检、设置保存均产生可见状态或弹窗回执。
