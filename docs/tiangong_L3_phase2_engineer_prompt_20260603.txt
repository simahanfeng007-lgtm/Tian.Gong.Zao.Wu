二、L3 第二阶段定位
────────────────────────────────

L3 第二阶段名称：

【Run / Task / Turn / Step 编排与连续性评分】

一句话定位：

本阶段只建立 L3 对 run、task、turn、step 的流程编排结构，以及暂停、等待、取消、失败、续接等流程连续性评分与状态转移建议；不做真实运行循环，不调用模型，不调用工具，不做权限裁决，不进入 Skill / ToolGroup / L4 / L5 / L6。

本阶段必须继承 L3 第一阶段的基础对象与数学驱动骨架，继续保持：

- 编排对象是纯数据 / 纯逻辑对象；
- 数学模型只输出评分建议；
- 情感系统只影响倾向权重；
- 动态驱动只影响排序和优先级；
- 所有外部动作只能表示为“请求对象”“引用对象”或“状态更新建议”；
- 最终执行未来仍必须经过 L5 边界层与 L4 执行层。

七、数学评分要求
────────────────────────────────

本阶段必须接入 L3 第一阶段数学骨架，但只做 run/task/turn/step 连续性相关的基础评分。

评分方向建议：

1. ContinuityIndex
   - 衡量流程前后是否连贯；
   - 关注 step 顺序、turn carryover、task 目标一致性、run 上下文完整性。

2. ResumabilityIndex
   - 衡量当前流程是否适合续接；
   - 关注是否有可恢复状态、是否有明确下一步、是否缺关键引用。

3. InterruptionSeverityScore
   - 衡量中断严重度；
   - 区分等待、暂停、取消、失败、阻断。

4. StepReadinessScore
   - 衡量下一 step 是否具备编排条件；
   - 只判断编排条件，不判断真实执行可行性。

5. ProgressCoherenceScore
   - 衡量 task / run 进度记录是否自洽；
   - 发现重复 step、倒退、跳跃、缺状态。

6. ContextCarryoverScore
   - 衡量 turn 间上下文续接价值；
   - 不做真实上下文拼接，只输出 carryover 建议。

7. RecoveryPriorityScore
   - 衡量恢复路径优先级；
   - 不执行恢复，只输出恢复建议。

8. CancellationSuitabilityScore
   - 衡量是否建议取消或放弃；
   - 不执行取消，只输出建议。

评分必须可解释。
评分必须输出依据。
评分不得成为权限裁决。
评分不得生成执行令。
评分不得跳过 L5。
评分不得触发 L4。
评分不得调用 L6。
