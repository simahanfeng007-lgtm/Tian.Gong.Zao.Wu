天工造物 / 临渊者 L6.36【失败恢复、可回放与质量门贯通】交付报告
生成日期：2026-06-06
基线来源：L6.35【执行 Step 状态机与长链压力基线】完整交付包

一、本轮目标

本轮继续执行“先完成执行链路全线贯通，再补接其他系统”的路线，只压实执行主链，不补接 Provider / Skill / Handoff / 情志 / GUI / 安装包等系统。

L6.36 目标是把 L6.35 的 Step 状态机与长链压力能力进一步贯通为：
- FailureTaxonomy：失败分类。
- RecoveryPlan：恢复计划。
- ReplayReport：可回放报告。
- QualityGateResult：执行链质量门结果。
- Replay Corpus：覆盖典型执行、失败、阻断、确认、交付、质量检查场景。

三、失败分类口径

L6.36 失败分类包含：
- model_plan_invalid
- risk_blocked
- confirmation_required
- tool_failed
- timeout
- budget_exhausted
- validation_failed
- delivery_failed
- skipped_after_stop

每类失败都会映射到 next_action，例如：
- risk_blocked → stop_for_safety_and_replan
- confirmation_required → ask_confirmation
- tool_failed → retry_or_generate_repair_plan
- timeout → split_step_or_retry_with_timeout_budget
- validation_failed → run_repair_then_quality_gate
- delivery_failed → repair_delivery_artifact_then_retry

四、质量门贯通口径

L6.36 的 QualityGateResult 不替代 L6.18 质量门，而是把执行链自身证据标准化：
- report_has_steps
- replay_events_reconstructable
- resume_envelope_present
- failure_taxonomy_complete

结果分为：
- pass：执行完成、可回放、恢复信封完整。
- warn：存在确认等待或跳过状态，允许续接但需披露。
- fail：存在可恢复失败、超时、验证失败或回放证据缺口。
- blocked：存在安全阻断或 P0 失败类型。

六、结论

L6.36 已把执行链从“能执行、能长链、能续接”推进到“失败可分类、恢复可计划、执行可回放、结果可质量门判断”。
这为 L6.37 执行链全线贯通冻结提供了前置证据。
