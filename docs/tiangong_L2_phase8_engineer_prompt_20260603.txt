────────────────────────────────
一、本阶段定位
────────────────────────────────

L2 是状态层。
第八阶段是 L2 的总收口阶段。

本阶段只做四件事：

1. 组件状态：
   定义 L2 内部组件、状态包、状态域、模块健康、版本、依赖、导出关系的状态对象。

2. 兼容迁移状态：
   定义状态 schema 兼容、旧状态映射、迁移提示、弃用提示、兼容风险的状态对象。
   注意：只记录迁移状态，不执行真实迁移。

3. 状态投影：
   定义面向人类、面向大模型、面向调试、面向审计、面向后续层的结构化状态投影对象。
   注意：只定义投影数据结构，不实现真实 UI、不生成自然语言聊天、不调用模型。

4. L2 总收口：
   整理 __init__.py 导出、状态总索引、全阶段兼容测试、冻结报告、未做事项、L3 交接边界。
   注意：只做 L2 层冻结收口，不启动 L3 开发。

这些对象只回答：

1. L2 有哪些状态组件？
2. 各状态组件版本、健康、依赖、导出是否清晰？
3. 状态 schema 是否存在兼容或迁移提示？
4. 某个完整生命体运行切片可以如何被投影给人、人类审计、大模型或后续层？
5. L2 全阶段是否可稳定序列化、稳定 hash、可快照、可增量、可冻结交接？
6. L2 与 L0/L1/L3-L6 的边界是否清楚？

这些对象不回答：

1. 后续 L3 应该如何调度；
2. 后续 L4 应该如何执行；
3. 后续 L5 应该如何裁决；
4. 后续 L6 子系统应该如何实现；
5. 如何真实迁移旧数据；
6. 如何真实投影到 UI；
7. 如何让模型实时读取或执行；
8. 如何调用工具、运行命令、访问文件、访问网络或执行插件。

第八阶段的核心边界是：

投影状态可以存在；
真实投影服务不得存在。
兼容迁移状态可以存在；
真实迁移执行不得存在。
组件状态可以存在；
真实组件宿主不得存在。
总收口可以存在；
L3 开发不得开始。

────────────────────────────────
四、核心对象设计要求
────────────────────────────────

所有状态对象必须满足：

1. 使用 @dataclass(frozen=True, slots=True)。
2. 字段必须可稳定序列化。
3. 字段默认值必须安全，不使用可变默认值。
4. 不持有真实文件句柄、数据库连接、网络连接、模型客户端、工具对象、回调函数、协程、线程、锁或运行时执行器。
5. 只记录 id、ref、status、kind、scope、timestamp、boundary、summary、hash、版本号、依赖、导出、投影片段、质量指标等状态事实。
6. 对可能很长的内容要克制。优先记录摘要、hash、引用、长度、来源、可见性，而不是保存全文。
7. 必须能参与快照、增量、稳定 hash、等值比较。
8. 必须有中文 docstring。
9. 必须与 L0/L1 类型边界保持兼容。
10. 禁止 import L3、L4、L5、L6 或任何未来层实现。
11. 禁止使用第三方依赖。
12. 禁止任何真实 IO、模型调用、工具调用或外部副作用。

建议字段类型优先使用：

- str
- int
- float
- bool
- tuple
- frozenset
- enum
- datetime 字符串或 L0 时间类型
- L0/L1 已定义的引用对象
- 本阶段状态对象

避免使用：

- dict 的可变嵌套结构；
- list；
- set；
- 任意 object；
- Callable；
- Any；
- 真实外部对象。

如必须使用 dict/list 输入，应在构造或工厂函数中规整为 tuple/frozenset 或稳定映射结构。

   - 不得执行检索；
   - 不得执行学习；
   - 不得生成 Skill。

7. 与第七阶段候选、变更、迭代、进化、实验、验证、恢复状态兼容：

   - 不得生成补丁；
   - 不得执行实验；
   - 不得执行验证；
   - 不得执行回滚。
