四、自由意志时间系统设计
1. FreeWillTimeTable 不创建后台线程。
2. Runtime 可在安全时机调用 build_tick。
3. interval_seconds 被硬限制在 180-900 秒，即 3-15 分钟。
4. 数学模型：
   - curiosity / achievement / order / low_rest 提高自主推进驱动力，使 interval 变短。
   - resource_pressure / failure_pressure / rest 提高压力，使 interval 变长。
5. tick 的 can_generate_goal 必须同时满足：
   - due=True。
   - autonomy_enabled=True。
   - 无活跃用户任务，或用户允许自治推进。
   - 无长链任务占用。
   - 无恢复优先任务。
   - budget_pressure < 0.88。
   - context_pressure < 0.88。

五、自主目标队列设计
每次有效 tick 最多生成候选目标，不执行目标。
目标类型：
1. learning：学习目标。
2. task：任务推进目标。
3. maintenance：维护目标。
4. review：复盘目标。
5. iteration：迭代候选目标。

目标字段包含：
- goal_id
- goal_type
- summary
- source_tick_id
- source_refs
- expected_value_score
- risk_level
- budget_estimate_score
- requires_confirmation
- status
- planner_hint
- no_direct_execution
- no_background_execution
- no_tool_invocation
- no_budget_mutation
- no_memory_write
- no_kernel_mutation

规则：
- A0-A2 可作为 Planner 候选。
- A3/A4 必须 requires_confirmation=True。
- A5 必须 blocked。
- 队列只排序候选，不执行候选。

六、自我迭代前端专区设计
新增 SelfIterationFrontendProjection，供前端展示“自我迭代区”。
展示内容：
- 发现的问题 / 沟通需求
- 建议改进方向
- 预计改动范围
- 风险等级
- 验证要求
- 回滚要求
- 是否需要用户确认

用户点击确认后，只生成 UserConfirmedIterationTicket。
该票据只允许 Planner 生成改造计划，不允许直接执行：
- permits_planner_draft_generation=True
- permits_execution=False
- requires_quality_gate=True
- requires_rollback_checkpoint=True
- requires_core_pollution_check=True
- no_patch_apply=True
- no_merge=True
- no_hot_switch=True
- no_core_mutation=True
