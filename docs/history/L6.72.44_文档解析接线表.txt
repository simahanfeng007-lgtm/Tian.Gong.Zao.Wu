三、Planner 分流规则
1. 自然语言直达 PlanBridge：
- “帮我读一下桌面的 xx.docx” -> document_parse(path=desktop_relative_path/xx.docx)
- “帮我总结下载里的 xx.pdf” -> document_parse(path=downloads_relative_path/xx.pdf)
- “打开这个 xlsx 看看内容” -> document_parse(path=xx.xlsx)
- “帮我看这个 pptx” -> document_parse(path=xx.pptx)

2. 模型计划 plan_schema 改写：
- read_file + .docx/.pdf/.xlsx/.pptx/.csv/.md/.txt/.html/.json/.py/.js/.ts/.tsx/.jsx/.css/.java/.cpp/.c/.go/.rs 等 -> document_parse
- safe_command_runner/safeCommandRunner + cat/type/more/less/Get-Content/open/start + 文档路径 -> document_parse

3. 禁止路径：
- docx/pdf/xlsx/pptx 不允许 shell 硬读。
- 文档读取失败不允许误报 Provider/API。
- raw bytes / stderr / tool raw result 不进入 ChatMessage.text。

四、解析方式
- txt/md/csv/json/code/html/xml/yaml/toml/sql：文本解码器，UTF-8/GB18030/CP936/UTF-16/Big5 优先。
- docx：python-docx 或 zip+xml 提取标题、段落、表格。
- xlsx：openpyxl 提取 sheet、表头、单元格摘要。
- pptx：python-pptx 或 zip+xml 提取每页文本。
- pdf：优先 pypdf/pdfplumber；依赖缺失时输出明确诊断，不输出乱码。
- 图片/zip/二进制：只显示类型、大小、路径、建议工具，不读原始字节。

五、风险等级
- document_parse：A1 只读/诊断类。
- safe_command_runner：仍走 Code-X 原生风险等级与 QualityGate；文档读取场景会被 Planner 改写为 document_parse。
- A5 Only 口径保持：A0-A4 默认放行，A5 才硬阻断。
