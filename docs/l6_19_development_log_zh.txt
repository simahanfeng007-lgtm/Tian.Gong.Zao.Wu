天工造物 / 临渊者 L6.19 产品交付系统标准化开发日志
生成日期：2026-06-05
基线：L6.18 测试验证质量门系统

一、阶段定位

L6.19 不是继续增加裸工具权限，而是把 L6.18 已有质量门、L6.17 工程诊断、审计摘要、运行报告线索和 ZIP 产物统一收束为标准 Release Bundle。
目标是让“可打包”升级为“可发布交付”：必须先有质量门裁决，再经过 Release Gate，最后生成带 Manifest、SHA256 和敏感扫描摘要的标准发布包。

二、核心新增

1. 新增 tiangong_agent_runtime.delivery_manifest
- DeliveryManifestBridge：保存最近一次交付 Manifest。
- ReleaseBundleBuilder：生成标准发布包。
- ReleaseGateVerdict：把质量门与敏感扫描统一成 pass / warn / blocked。
- SecretScanFinding：记录敏感路径、凭证路径和明显密钥内容信号。

2. 新增 create_release_bundle Runtime Adapter
- 工具风险等级：A3。
- 必须走 ExecutionSpine / RiskClassifier / PermitGateway / AuditBridge。
- 不允许绕过质量门直接发布。
- quality_gate 为 fail / blocked 或 allow_package=False 时阻断发布 ZIP。
- 发现 .env、key、token、credential、secret、私钥后缀或敏感目录时阻断发布 ZIP。

3. 新增 RuntimeEntry.run_release
固定流程：
scan_project → compileall → pytest（默认要求） → diagnose_project → evaluate_quality_gate → write_quality_report → create_release_bundle。
其中 L6.18 质量门阶段不再调用旧式 create_zip_package，标准发布统一交给 L6.19 Release Bundle。

4. 新增 CLI 命令
- /delivery：显示最近交付 Manifest。
- /delivery-save <path>：导出交付 Manifest JSON。
- /delivery-reset：清空交付 Manifest。
- /release [zip] [path]：执行标准发布流程。

三、标准发布包内容

Release Bundle ZIP 内包含：
- RELEASE_MANIFEST.json
- RELEASE_MANIFEST.md
- reports/quality_gate.json
- reports/diagnosis.json
- reports/audit_summary.json
- payload/<源文件>

ZIP 外侧车文件：
- <release>.zip.sha256
- <release>.zip.manifest.json

说明：ZIP 内 Manifest 不能自包含最终 ZIP 自身 SHA256，否则会形成自引用哈希问题；最终 SHA256 写入侧车 Manifest 与 .sha256 文件。

四、边界

- 未修改 tiangong_kernel。
- 未开放任意 shell。
- 未开放裸模型 SDK。
- 未跳过 PermitGateway。
- 未把敏感文件打入发布包。
- 未声明 full pytest 全量通过。
