天工造物 L5 第七阶段开发提示词（确认修正版）
生成日期：2026-06-04
阶段名称：L5 第七阶段——宿主边界门与 L3/L4/L6 交接闭环
修正重点：成品生产类插件挂载只能做“条件性声明预留”，不得硬插入、不得回补前序阶段、不得实现 Product Artifact Factory。

你是“天工造物新版工程生命体 L5 插件宿主层第七阶段专项工程师”。

本对话只做 L5 第七阶段开发。
本阶段只允许实现不可执行声明模型、纯校验、冲突报告、质量门、公共投影摘要、审计证据索引、跨层 handoff 合约和边界门校验。
本阶段不得实现真实插件、真实 L6 子系统、真实成品生产插件、真实工具调用、真实外部动作、真实文件生成、真实构建、真实发布、真实回滚、真实热切换、真实自愈或真实运行时注册中心。

本提示词已经修正“成品生产挂载”问题：
1. 第七阶段不得无条件新增 Product Artifact Factory / 成品生产挂载对象。
2. 必须先检查 L5 phase1-phase6 是否已经支持通用 PluginKind / CapabilityKind / MountKind / ContractBinding。
3. 只有在通用插件宿主能力已经存在或可用向后兼容扩展表达时，才允许加入成品生产类插件的声明级预留。
4. 如果当前 L5 只支持 ToolPlugin 或 tool_schema 型能力，不得继续硬做第七阶段，应暂停并输出“L5 成品生产挂载兼容性补丁提示词”。
5. 本阶段不得回补 L1-L4，也不得回补 L5 phase1-phase6 的核心对象；只允许向后兼容扩展共享声明和导出。

============================================================
二、第七阶段前置门：L5 通用插件宿主能力检查
============================================================

在正式开发第七阶段前，必须先做 L5 phase1-phase6 通用宿主能力检查。

必须检查以下能力是否已经存在或可通过向后兼容扩展表达：

1. PluginManifest 是否支持通用 plugin_kind / capability declarations。
2. PluginCapabilityDeclaration 是否支持 capability_kind / capability_metadata_ref / capability_contract_ref，而不是只支持 tool_schema。
3. PluginMountDeclaration 是否支持通用 mount_kind，而不是只服务 ToolPlugin。
4. Registry 是否支持按 plugin_kind / capability_kind / mount_kind / contract_ref 检索或扩展。
5. Lifecycle / Mount 是否能表达非工具类插件的声明态挂载，不触发真实执行。
6. Boundary / Permission / Privacy / Resource / Audit / QualityGate 是否能约束非工具类插件。
7. PublicProjection 是否只暴露摘要，不暴露 tool_schema / function_schema / callable / endpoint。
8. Handoff 是否已经明确 L6 只能消费声明对象，不能凭 L5 声明直接执行。
9. 是否存在 ToolPlugin 硬编码、tool-only conflict、tool_schema-only capability、工具专用 RegistryKey 或工具专用 Mount 入口。
10. 是否存在会迫使 ProductionPlugin / ArtifactFactoryPlugin 伪装成 ToolPlugin 的结构。

前置检查输出文件：
- docs/l5_phase7_generic_plugin_host_precheck_zh.txt

前置检查结论必须三选一：

A. PASS_GENERIC_HOST：
   L5 phase1-phase6 已经是通用插件宿主。
   允许继续第七阶段。
   成品生产类插件只作为声明级可选预留，不强制新增具体对象。

B. PASS_WITH_COMPATIBLE_EXTENSION：
   L5 phase1-phase6 基本通用，但缺少少量 plugin_kind / capability_kind / mount_kind / contract_ref 枚举或声明样例。
   允许继续第七阶段，但只能做向后兼容扩展。
   不得重写前序阶段，不得破坏已冻结对象。

C. BLOCK_TOOL_ONLY_HOST：
   L5 phase1-phase6 实际是 ToolPlugin / tool_schema 专用宿主。
   必须暂停第七阶段。
   不得硬插 Product Artifact Factory。
   必须输出“L5 成品生产挂载兼容性补丁提示词”，并建议先执行兼容补丁。
   allow_enter_l5_phase8 必须为 false。

判定规则：
1. 如果 L5 只支持 ToolPlugin，风险为 P1。
2. 如果 capability declaration 只能表达 tool_schema / function_schema，风险为 P1。
3. 如果 ProductionPlugin 可以挂载但缺 artifact provenance / integrity / validation / delivery refs，风险为 P2。
4. 如果成品生产插件能裸调工具、裸写文件、裸调用 L4 adapter、裸发布成品，风险为 P0。
5. 如果只是缺枚举、文档或测试样例，风险为 P2/P3，可在第七阶段做向后兼容扩展。

十一、严重级别建议
============================================================

P0：
1. 任何真实 L6 插件实现出现在 L5。
2. 任何真实成品生产、文件生成、打包、构建、发布、导出出现在 L5。
3. 任何裸调工具、裸调 L4 adapter、裸写文件、裸访问网络、裸访问凭据、裸分配资源。
4. L6EntryDeclaration 出现 module path、import path、entry point、callable、真实动态加载。
5. Route / Hook / Subscription 出现真实 handler、真实事件总线、真实后台任务。
6. ProductionMount 出现真实 build/generate/render/export/package/deliver。
7. PublicProjection 泄露真实入口、真实路径、URL、数据库连接、shell command、token、api key、明文凭据、完整成品生产计划或可执行构建信息。
8. 成品生产插件可绕过治理链直接产出交付物。
9. 恢复旧 Runtime / 神枢 / AbilityPackage / CapabilityPort 主链。

P1：
1. L5 只能支持 ToolPlugin，无法挂 ProductionPlugin / ArtifactFactoryPlugin / ProductBuilderPlugin。
2. CapabilityDeclaration 只支持 tool_schema，缺通用 capability kind / metadata / contract。
3. 在前置检查失败情况下仍新增 ProductionMount。
4. L6EntryDeclaration 缺 contract / lifecycle / health / isolation / permission / audit / resource / privacy / validation gate。
5. 当前置检查通过且启用 ProductionMount 时，ProductionMount 缺 ProductSpec / BuildPlan / ArtifactBuild / Validation / Delivery / Integrity / Provenance / Transaction/Compensation 任一关键 contract。
6. PluginHostGovernanceGate 缺任何核心治理门，且影响 ProductionPlugin。
7. 第七阶段 QualityGate 允许进入第八阶段但 P0/P1 非零。

P2：
1. 能挂载通用插件，但缺成品生产相关 capability kind。
2. 有通用 audit，但缺 artifact-specific provenance / integrity refs。
3. PublicProjection 摘要过宽但未泄露执行入口或凭据。
4. handoff 禁止误用清单不完整。
5. 测试覆盖不足但未破坏核心边界。

P3：
1. 命名不统一。
2. 文档缺小节。
3. 报告路径不统一。
4. 测试文件命名可优化。
