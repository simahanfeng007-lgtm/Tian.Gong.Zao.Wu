# 临渊者 L6.18 测试验证质量门系统说明

## 定位

L6.18 把“能跑测试”升级为“能做质量门裁决”。它不是新开执行后门，而是在现有受治理 Runtime 链上新增质量门桥：扫描、质量检查、诊断、质量门裁决、报告、按裁决决定是否打包。

## 新增命令

```text
/quality-gate [path]
/quality-report
/quality-save <path>
/quality-reset
```

## 裁决结果

- `pass`：允许继续交付。
- `warn`：允许受控打包，但必须披露 P2 警告。
- `fail`：允许继续修复，不允许发布打包。
- `blocked`：停止自动链路，先处理治理阻断或高危问题。

## 执行链

```text
scan_project
→ run_python_quality_check(compileall)
→ run_python_quality_check(pytest，可按策略要求)
→ diagnose_project
→ evaluate_quality_gate
→ write_workspace_file(report)
→ create_zip_package（仅 allow_package=True 时）
```

所有步骤仍走：

```text
ExecutionSpine
→ RiskClassifier
→ PermitGateway
→ RuntimeToolRegistry
→ Adapter
→ AuditBridge
```

## 安全边界

- 不修改 `tiangong_kernel/` 主体。
- 不开放任意终端命令。
- 不读取完整源码做公共投影。
- 不记录 API Key。
- 不绕过 registry / adapter / audit。
- 质量门失败时不会自动打包发布。
