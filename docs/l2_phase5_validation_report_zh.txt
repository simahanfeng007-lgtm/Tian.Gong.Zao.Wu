三、测试覆盖结论

1. import / exports

八个第五阶段新增模块均可直接 import；`tiangong_kernel.l2_state.__init__` 已导出第五阶段公开对象；第一至第四阶段导出未被破坏。

2. frozen slots

第五阶段 8 个公开状态 dataclass 均为 `@dataclass(frozen=True, slots=True)`；实例化后字段不可变。

3. stable serialization / stable hash

第五阶段状态对象均可被 `stable_json_dumps` 和 `stable_hash` 处理；同一对象多次序列化结果一致；hash 长度为 64。

4. ObservationSourceState

可表达 model_output / tool_result / boundary_event / test_event 等来源类型；可表达 available / stale / redacted / unknown 等来源状态；可引用 boundary、security、environment、run、task、skill、tool group、tool intent、action 和 effect。

5. ObservationChannelState

可表达 direct_state_input / event_projection / audit_projection / metric_projection；可表达 open / interrupted / closed / expired；不提供订阅、发布、消费、监听或轮询方法。

6. ObservationFrameState

可表达 event / metric / audit / effect / model_feedback / boundary / security 等帧类型；可表达 partial / conflicted / redacted / stale；payload 只保存引用，短摘要超过 512 字符会被拒绝。

7. EventStreamState

可表达 streaming / interrupted / truncated / closed；稳定保存 frame_count、dropped_frame_count、redacted_frame_count；拒绝负计数；不实现事件总线、网络流或真实事件消费。

8. ObservationMetricState

可表达 latency / token_usage / resource_pressure / health_signal；稳定保存 metric_value_repr；不提供采样或系统指标读取方法。

9. AuditObservationState

可表达 run / task / skill / tool / action / effect / boundary / security 审计观察；audit_payload_ref 只保存引用；短摘要超过 512 字符会被拒绝；不提供写审计、签名或验签方法。

10. ObservationQualityState

可表达 completeness / freshness / consistency / redaction_safety / conflict_level；可表达 good / partial / conflicted / unsafe / unknown；不提供质量评分、可信度计算或冲突解决方法。

11. ObservationProjectionState

可表达 run / task / skill / tool / boundary / security / quality / audit 观察投影；投影保存结构化短摘要和引用；不生成聊天文本、最终模型提示词或下一步行动。

12. 跨阶段链路

构造并验证了纯状态链：
RunState → SkillState → ToolIntentState → ActionIntentState → EffectObservationState → ObservationFrameState → ObservationQualityState → AuditObservationState → ObservationProjectionState。

同时连接第四阶段 BoundaryCheckState、ResourceBudgetState、EnvironmentState、SecurityBoundaryState。链路仅通过 `TypedRef` / `state_ref` 表达。

13. 源码边界扫描

第五阶段新增模块未导入 os、pathlib、socket、subprocess、requests、urllib、httpx、threading、multiprocessing 等真实 IO / 网络 / 进程 / 后台任务依赖；未定义 observe、watch、listen、collect、sample、poll、read_log、read_metric、trace、emit、write_audit、stream、subscribe、consume、execute、invoke、call_model、call_tool、recover 等业务方法；未导入 L3 / L4 / L5 / L6 或旧 Runtime / 旧能力包 / 插件宿主模块。
