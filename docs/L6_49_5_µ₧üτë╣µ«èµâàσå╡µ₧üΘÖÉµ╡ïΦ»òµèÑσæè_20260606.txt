====================================================

一、本轮目标

基于 L6.49.4 连续 10 轮 Runtime 接线轨迹压测基线，追加 20 种极特殊/异常/边界场景，验证：

1. Runtime 情志 previous_state 是否持续演化，不回退为“第一天”。
2. 遗忘审查是否稳定走 review(record, vector)，不静默吞错、不物理删除。
3. 记忆召回是否只读摘要级投影，不裸写长期记忆，不暴露原始密钥。
4. 生命周期、四路径、预算、Planner 消费层是否保持只读候选/投影边界。
5. 计划解析与执行链是否对危险路径、shell 注入、非法工具 fail-safe。
6. external_context_hint 是否会把 api_key/token/endpoint 原文泄露给模型计划器。
7. MemoryStore replay / snapshot 是否稳定可复现，避免 hash compare 漂移。

三、发现问题

【P1】external_context_hint 敏感信息可能进入 ModelPlanner provider call

现象：
- external_context_hint 中若包含 api_key=... / token=... / base_url=... / Authorization: Bearer ...，旧逻辑会原样拼入 planner context。
- 若 planner_mode=MODEL_SUGGEST，这些原文会发送给模型计划器。

修复：
- tiangong_agent_shell/safe_logging.py：增强 redact_text，支持无 known secret list 时的显式敏感字段脱敏：
  - api_key / apikey / token / secret / password / credential
  - Authorization: Bearer ...
  - base_url / endpoint / endpoint_url / provider_base_url
  - sk-... 形态密钥
- tiangong_agent_runtime/runtime_entry.py：_build_planner_context_hint 先对 external_context_hint 执行 redact_text，再进入 Planner 上下文。

边界：
- 不屏蔽普通 URL，避免破坏正常网页/文档任务。
- 只对明确带敏感字段名的 endpoint/base_url 做摘要态替换。

【P2】MemoryStore replay 的 updated_at 非确定，导致 snapshot/hash 漂移

现象：
- MemoryRecord.with_updates() 默认 updated_at=time()。
- MemoryStoreBridge.replay_records() 重放 tombstone / suppress / use_feedback 时会触发当前时间写入状态对象。
- 同一 JSONL 事件流，多次 export_snapshot() 得到不同 updated_at，容易被误判为 MemoryStore 被压测改写，也会影响 hash compare 稳定性。

修复：
- MemoryRecord.with_updates() 默认保留原 updated_at，除非调用方显式传入 updated_at。
- replay_records() 对 tombstone / suppress / use_feedback 使用事件 created_at 作为 updated_at / last_accessed_at。

边界：
- 未改变 append-only 事件模型。
- 未改变 MemoryStore 真实写入接口。
- 修复的是 replay 投影确定性，不是新增记忆功能。

四、关键验证结果

1. 20 个极特殊场景全部通过。
2. 情志 turn_count 从 1 连续增长到 20。
3. 情志 risk_attention 在 20 场景中出现多档波动；A5/密钥泄露与 shell 注入文本均明显抬高风险关注。
4. emotion_temporary_delta 未归零，跨 20 轮持续变化。
5. 遗忘审查每轮 review_count=10，稳定执行 limit=10。
6. L5 保护记忆遇到用户遗忘请求时触发 retention_exception_review_required，不允许 direct_delete。
7. tombstoned / suppressed 记忆未进入 Runtime 遗忘审查。
8. MemoryStore event_count 压测前后保持 19，事件流未变化。
9. external_context_hint 中原始 key/token/endpoint/bearer 未进入 ModelPlanner provider call。
10. plan_schema 对 output.steps / function_call / input alias 可接受，对 ../.env、rm -rf、shell 工具 fail-safe。
