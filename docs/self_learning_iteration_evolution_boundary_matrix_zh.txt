一、引用关系

1. LearningRef：学习信号或学习结果引用，不等于候选。
2. AdaptationRef：适应性调整引用，不等于执行。
3. EvolutionRef：结构级演进引用，不等于提交。
4. ImprovementProposalRef：改进提议引用，不等于 patch。
5. ImprovementAssessmentRef：评估引用，不等于批准。
6. EvolutionCommitRef / EvolutionCommitIntent：提交意图引用，不等于提交。
7. EvolutionRollbackRef / RollbackValidationHint：回滚与回滚验证引用，不等于真实回滚。

二、状态流

Candidate
→ Evidence
→ Validation / Experiment
→ CandidateValidatedToCommitAdvice
→ L5 permit + human confirmation when required
→ L4 requirement-only handoff
→ L6 future commit / activation / observation service
→ PostCommitObservation
→ RollbackValidation or TombstoneMigration when needed

三、禁止误读

1. Candidate 不是 patch。
2. Validation 不是批准。
3. DecisionHint 不是 permit。
4. CommitIntent 不是 commit。
5. ActivationHint 不是 runtime activation。
6. HotSwitchBoundary 不是 hot switch。
7. RollbackValidationHint 不是 rollback。
8. TombstoneHint 不是物理删除。

四、L5/L6 未来职责

L5：
1. Commit permit。
2. Hot switch permit。
3. Rollback permit。
4. Human confirmation。
5. Audit gate。
6. Lease and scope match。

L6：
1. Learning sink。
2. Validation runner。
3. Experiment runner。
4. Patch generator。
5. Commit service。
6. Hot switch service。
7. Rollback service。
8. Observation monitor。

所有 L6 能力必须经 L5 边界治理后通过 L4 action grounding 承载，不得裸执行。
