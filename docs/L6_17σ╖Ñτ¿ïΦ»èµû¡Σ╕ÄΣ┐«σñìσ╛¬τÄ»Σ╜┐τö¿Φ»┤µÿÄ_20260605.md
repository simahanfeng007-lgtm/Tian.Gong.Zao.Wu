# 天工造物 L6.17 工程诊断与修复循环使用说明

## 定位

L6.17 在 L6.16 项目雷达基础上，新增工程诊断与修复循环底座。它不是任意代码自动改写器，而是 LLM 装甲的受治理诊断/复测/报告/打包闭环。

## 新增命令

```bash
python run_agent.py --mock --tool-mode runtime_governed --workspace ./workspace
```

交互命令：

```text
/diagnose .
/diagnosis
/diagnosis-save diagnosis.json
/repair-loop .
```

## 执行链

`/diagnose` 默认执行：

```text
scan_project -> run_python_quality_check(compileall) -> diagnose_project
```

`/repair-loop` 默认执行：

```text
scan_project -> compileall -> pytest -> diagnose_project -> write_workspace_file(report) -> create_zip_package
```

所有步骤仍走：

```text
ExecutionSpine -> RiskClassifier -> PermitGateway -> RuntimeToolRegistry -> Adapter -> AuditBridge
```

## 安全边界

- 不修改 `tiangong_kernel/` 主体；
- 不开放任意 shell；
- 不开放桌面控制；
- 不读取敏感文件内容；
- 不把 API Key、完整 prompt、完整源码写入诊断；
- 诊断报告只包含安全摘要；
- 修复循环写报告和打包均通过既有受治理 adapter。

## 当前限制

- L6.17 不做任意代码自动补丁；
- 不做长期记忆写入；
- 不做多智能体调度；
- 不做真实 Provider 全联调；
- 不做前端或安装包。
