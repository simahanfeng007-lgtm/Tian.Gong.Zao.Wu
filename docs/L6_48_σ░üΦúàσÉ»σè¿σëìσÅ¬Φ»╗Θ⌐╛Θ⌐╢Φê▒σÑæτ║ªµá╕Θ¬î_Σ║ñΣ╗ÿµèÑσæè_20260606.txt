================================================================

一、本轮定位

本轮不是重新规划，也不是接入真实桌面端前端。当前包体未发现独立 frontend / desktop / electron 源码目录，只有 CLI 外壳与 Runtime 公共投影。因此本轮按“封装启动前核验”处理：

1. 保持 RuntimeEntry / PlannerExecutionController / ExecutionSpine 执行主链不变。
2. 在既有 public_projection_bridge.py 中补充桌面端可读的只读驾驶舱投影契约。
3. 修复 CLI 状态与配置输出中 endpoint / key 仍可能展示部分原文的问题，统一改为 digest 态。
4. 增加 L6.48 回归测试，验证前端未来只能读取 projection，不获得工具调用、Provider SDK、记忆写入、自我迭代合入口。

二、本轮发现

P0：未发现。
P1：发现 1 个封装前敏感信息显示风险。
- 原状态：CLI status / show-config 可展示 base_url 原文，api_key 采用前后缀遮罩。
- 风险：与“endpoint、token、key 等敏感信息只保存 digest，不展示原文”的前端/封装口径不一致。
- 修复：base_url / endpoint / api_key 均改为 <已配置:digest:xxxx>。
