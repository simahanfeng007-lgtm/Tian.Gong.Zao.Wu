────────────────────────────────

阶段名称：

本阶段目标：

在 L3 第一阶段“编排基础骨架与数学驱动基础骨架”、L3 第二阶段“Run / Task / Turn / Step 编排与连续性评分”的基础上，新增 Skill 直显流程、Skill 选择流程、Skill 激活流程、ToolGroup 解析流程、ToolGroup 释放流程、ToolLease 编排建议，以及相关数学评分对象。

L3 第三阶段负责“如何把 Skill 直显与 ToolGroup 释放流程组织成可审查、可排序、可续接、可记录的编排建议”，不负责“Skill 从哪里来、工具如何真实释放、是否允许释放、工具如何执行”。

七、数学驱动要求
────────────────────────────────

本阶段必须纳入数学驱动，但只能做轻量评分骨架和建议，不实现复杂策略。

输入方向：

1. L2 状态切片引用；
2. L3 Run / Task / Turn / Step 编排上下文；
3. Skill 候选引用；
4. ToolGroup 候选引用；
5. 目标向量；
6. 约束集合；
7. 情感权重输入引用；
8. 动态驱动权重输入引用；
9. 连续性评分引用；
10. 稳定性/可逆性/最小暴露约束。

评分方向：

1. SkillMatchScore：Skill 与任务目标匹配度；
2. SkillContinuityScore：Skill 与当前 Run/Task/Turn/Step 连续性；
3. SkillReadinessScore：Skill 是否具备进入后续流程的准备度；
4. ToolGroupNeedScore：是否需要某个 ToolGroup；
5. ToolGroupMinimalityScore：释放工具组是否最小充分；
6. ToolExposureCostScore：工具暴露成本；
7. ToolGroupSufficiencyScore：工具组是否足够支撑后续意图；
8. ReversibilityIndex：后续路径可逆性；
9. StabilityIndex：流程稳定性；
10. OrchestrationRanking：候选路径排序。

输出方向：

1. MathEvaluation；
2. MathRecommendation；
3. SkillToolRouteRanking；
4. ToolGroupReleaseRanking；
5. StateTransitionAdvice；
6. SkillSelectionAdvice；
7. ToolGroupReleaseAdvice；
8. Explanation / reason_codes。

硬边界：

1. 数学评分只给建议；
2. 数学评分不做权限裁决；
3. 数学评分不做风险放行；
4. 数学评分不授予工具；
5. 数学评分不触发执行；
6. 数学评分不绕过 L5；
7. 数学评分不替代 L4；
8. 数学评分不实现 L6 专业算法。
