────────────────────────────────
一、阶段名称与目标
────────────────────────────────

阶段名称：

L3 第四阶段：ModelIntent / ToolIntent / ActionIntent 编排评分

本阶段目标：

在 L3 第一阶段“编排基础骨架与数学驱动基础骨架”、L3 第二阶段“Run / Task / Turn / Step 编排与连续性评分”、L3 第三阶段“Skill 直显与 ToolGroup 释放编排评分”的基础上，新增模型意图、工具调用意图、动作意图的编排对象、意图结构校验、参数完整度评分、执行准备度评分、意图缺口建议、拒绝/降级/澄清路径建议、意图到后续边界/执行阶段的非执行型衔接提示。

本阶段只做“意图编排”和“评分建议”。
本阶段不调用模型，不调用工具，不执行动作，不做 L5 边界裁决，不生成真实执行请求，不进入真实执行循环。

一句话边界：

L3 第四阶段负责“把模型/工具/动作意图整理成可检查、可评分、可排序、可续接、可交给后续边界层审查的编排建议”，不负责“模型如何产生意图、工具是否允许调用、动作是否执行、边界是否放行”。

七、数学驱动要求
────────────────────────────────

本阶段必须纳入数学驱动，但只能做轻量评分骨架和建议，不实现复杂策略。

输入方向：

1. L2 状态切片引用；
2. L3 Run / Task / Turn / Step 编排上下文；
3. Skill 选择与 ToolGroup 释放编排上下文；
4. ModelIntent / ToolIntent / ActionIntent 候选引用；
5. 目标向量；
6. 约束集合；
7. 情感权重输入引用；
8. 动态驱动权重输入引用；
9. 连续性评分引用；
10. Skill / ToolGroup 评分引用；
11. 稳定性/可逆性/最小暴露约束。

评分方向：

1. IntentCompletenessScore：意图字段和参数完整度；
2. IntentCoherenceScore：意图与上下文一致性；
3. IntentAmbiguityScore：意图歧义程度；
4. IntentReadinessScore：是否建议进入后续边界/执行准备流程；
5. IntentContinuityScore：意图与当前 Run/Task/Turn/Step 连续性；
6. IntentReversibilityScore：意图后续路径可逆性；
7. IntentDegradeScore：是否建议降级；
8. IntentClarificationNeedScore：是否建议澄清；
9. ToolIntentParameterCompletenessScore：工具意图参数完整度；
10. ActionIntentReadinessScore：动作意图准备度；
11. OrchestrationRanking：候选意图排序。

输出方向：

1. MathEvaluation；
2. MathRecommendation；
3. IntentRouteRanking；
4. IntentValidationAdvice；
5. IntentClarificationAdvice；
6. IntentDowngradeAdvice；
7. IntentRejectPathAdvice；
8. StateTransitionAdvice；
9. Explanation / reason_codes。

硬边界：

1. 数学评分只给建议；
2. 数学评分不做权限裁决；
3. 数学评分不做风险放行；
4. 数学评分不授予工具；
5. 数学评分不触发执行；
6. 数学评分不生成真实 BoundaryCheckRequest；
7. 数学评分不生成真实 ExecutionRequest；
8. 数学评分不绕过 L5；
9. 数学评分不替代 L4；
10. 数学评分不实现 L6 专业算法。
