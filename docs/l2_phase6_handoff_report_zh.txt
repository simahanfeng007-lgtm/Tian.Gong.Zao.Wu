四、对象作用与边界

Memory 系列状态：记录记忆引用、层级、可见性、召回结果引用、注入事实和健康计数；不执行记忆库读取、召回、遗忘、晋升、embedding、检索或 Skill 生产。

Context 系列状态：记录上下文片段、窗口、预算、压缩引用、注入事实和连续性；不拼接 prompt，不执行压缩，不计算真实 token，不写入对话历史，不调用模型。

Retrieval 系列状态：记录检索通道、请求、查询、结果引用、覆盖和质量；不执行文件搜索、网页搜索、数据库查询、向量检索、embedding、重排或 RAG。

Learning 系列状态：记录学习信号、学习需要、材料引用、准备度、边界和可见性；不实现学习算法，不生成 Skill、Tool、补丁、实验、验证、回滚或进化策略。

KnowledgeReferenceState：记录知识引用、来源、类型、hash、摘要、可见性、新鲜度和可信度；不抽取知识，不写知识库，不生成 SkillSeed、SkillVersion、SkillPatch 或 Tool 需求。

五、前置阶段接线

- MemoryRefState 可引用 Phase5 ObservationFrameState，并连接 Phase3 Run / Task / Skill 状态引用。
- ContextSegmentState 可引用 Phase3 Skill 状态引用，ContextWindowState 可引用 Phase3 ModelRequestState。
- RetrievalResultRefState 可引用 Phase5 ObservationFrameState。
- LearningSignalState 可引用 Phase5 ObservationQualityState 和 ObservationFrameState。
- L2StateSnapshot 可纳入第六阶段状态引用集合。

所有接线只使用 `TypedRef` / `identity.state_ref`。
