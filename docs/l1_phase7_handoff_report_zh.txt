13. 自我学习 / 自我迭代 / 自我进化端口说明
自我学习端口只提交候选、知识摄入意图、Skill 学习提示、证据、复核和边界声明，不启动学习。
自我迭代端口只提交迭代候选、补丁意图、复核、回滚提示、证据和边界声明，不生成补丁、不改文件、不合入。
自我进化端口只提交进化意图、候选、边界、证据、决策提示、回滚提示和连续性声明，不修改架构、不生成插件、不生产工具。

14. 与第五阶段、第六阶段的衔接说明
第七阶段复用第五阶段 Skill / ToolGroup 缺口对象：SkillGapReport、ToolNeedReport、ToolGroupGapReport、SkillEvolutionHint、SkillIterationHint、SkillCorrectionHint。
第七阶段复用第六阶段模型反馈 / 反思对象：ModelLearningIntent、ModelFailureFeedback、ModelCorrectionHint、ModelSkillGapFeedback、ModelToolNeedFeedback、ModelReflection、ModelOutcomeAssessment、ModelEvolutionHint、ModelIterationHint。
这些对象只作为候选来源和证据入口，不触发真实动作。

15. 为什么本阶段没有实现真实学习、迭代、进化
第七阶段是端口协议层，只负责表达边界、输入、输出、候选和证据。真实学习、迭代、进化会触及系统状态、文件、Skill 版本、工具生产、合入和回滚，必须留给后续层在验证、复核、边界和回滚协议下处理。
