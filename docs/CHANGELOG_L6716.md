# FE01 STEP31Q / L6.71.7

## 目标

修复真实 Provider 联调后的错误态提示，并把高频外观选择「永夜 / 极昼」下沉到底部状态栏，减少用户进入设置页的次数。

## 修改

1. Provider readiness 增加真实链路错误分类：网关不可达、API Key 无效、模型不存在、Provider 超时、限流、服务端错误、通用联调失败。
2. 本地桌面桥接在 provider 模式下记录最近一次真实模型链路结果，只输出脱敏错误码、脱敏摘要和下一步动作。
3. 设置页「检查状态」显示 last_check、last_error 与可执行下一步，不裸调 Provider SDK。
4. 底部状态栏增加「主题：永夜 / 极昼」快速切换按钮。
5. 设置页外观文案同步为永夜 / 极昼 / 墨绿，保留墨绿在设置页选择，不占用底框。

## 边界

- 前端不直接调用 Provider SDK。
- 前端不执行工具。
- 前端不写记忆。
- 前端不写审计。
- API Key / Base URL 仍只走 Runtime / 本地桥接托管，UI 只显示 digest 和错误码。
