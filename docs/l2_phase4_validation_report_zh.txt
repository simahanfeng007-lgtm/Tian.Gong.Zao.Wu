三、测试覆盖结论

1. import / exports

六个第四阶段新增模块均可直接 import；`tiangong_kernel.l2_state.__init__` 已导出第四阶段公开对象；第一、第二、第三阶段导出未被破坏。

2. frozen slots

第四阶段 22 个公开状态 dataclass 均为 `@dataclass(frozen=True, slots=True)`；实例化后字段不可变；集合字段默认保持 tuple。

3. stable serialization / stable hash

第四阶段状态对象均可被 `stable_json_dumps` 和 `stable_hash` 处理；同一对象多次序列化结果一致；hash 长度为 64；安全相关序列化只包含引用、脱敏标记和缺值标记。

4. ControlState

ControlPlaneState 可表达 READY / ACTIVE / PAUSED / BLOCKED / DEGRADED / LIMITED / CLOSED；可引用 RunState、TaskState、ToolIntentState、BoundaryState、ResourceState、SecurityState、ModelFeedbackState；ControlPlaneMode 仅作为模式标签；ControlSignalState 和 ControlConstraintState 均为记录型状态。

5. BoundaryState

BoundaryCheckState 可表达 NOT_REQUIRED / PENDING / PASSED / BLOCKED / DEGRADED / ALTERNATIVE_AVAILABLE / CONFIRMATION_REQUIRED / EXPIRED；可记录被检查对象、边界、风险视图、裁决、策略、证据、审计引用；BoundaryBlockedState、BoundaryDegradedState、BoundaryAlternativeState 分别覆盖阻断、降级和替代路径事实。

6. RiskDecisionState / A5

RiskDecisionState 可记录 RiskView 引用、Decision 引用、severity 标签和 policy 引用；未提供 score / calculate / decide / allow / deny 等方法；A5 仅作为序列化标签和状态引用出现，不触发 L2 内部硬拦截。

7. PolicyReferenceState

PolicyReferenceState 可记录策略引用、短策略名、版本引用、适用对象和来源边界；拒绝多行策略正文形态；不执行策略匹配。

8. ResourceState

ResourceBudgetState、QuotaState、RateLimitState、ResourceLeaseState、ResourcePressureState 可表达预算、配额、限速、租约和压力状态；未提供 sleep / retry / renew / degrade 等执行方法；不读取真实资源。

9. Environment / Sandbox

EnvironmentState、SandboxState、ExternalWorldReferenceState 可记录环境、沙箱和外部世界引用；源码扫描确认未导入 os、platform、socket、subprocess、pathlib 等真实探测或 IO 依赖。

10. Security / Privacy / Credential

SecurityBoundaryState 可记录安全边界、信任边界、隐私状态、凭据状态、边界检查和风险裁决引用；PrivacyCredentialState 与 SecretReferenceState 默认并强制 redacted=True、value_absent=True；源码与序列化结果均不包含禁止的真实敏感值字段名。

11. 第三阶段集成引用

BoundaryCheckState 引用 ToolIntentState；RiskDecisionState 引用 ActionIntentState；ResourceLeaseState 引用 ToolGroupReleaseState；SecurityBoundaryState 引用 EffectObservationState；ControlPlaneState 引用 ModelFeedbackState 和 ToolIntentState。引用链均使用 TypedRef。

12. 边界扫描

第四阶段新增模块未定义执行、调用、调度、恢复、风险评分、裁决、环境探测、安全扫描、凭据读取、预算扣减、配额预留等公开方法；未导入 L3 / L4 / L5 / L6；未导入旧 Runtime、旧能力包、插件宿主或真实适配器。

四、L0 / L1 状态
