三、关键设计

1. ScoreGuard
- Python 中 bool 是 int 子类，原先 isinstance(value, (int, float)) 会让 True/False 被当作 1/0。
- L6.40 将 score 校验统一为 ensure_score：拒绝 bool、非数值、NaN、越界值。
- 已覆盖 memory / affective / lifecycle 三类典型入口。

2. MemoryMathCore
- MemoryLevel：L1 / L2 / L3 / L4 / L5。
- MemoryCategory：working / episodic / semantic / procedural / self / runtime。
- DecayKernel：时间衰减 + reuse 强化 + success 强化。
- RecallScoreVector：召回分数与进入 PlannerContext 的硬过滤。
- PromotionScoreVector：晋升分数、低置信/隐私/污染/冲突阻断。
- ForgettingScoreVector：遗忘分数与用户显式忘记强制复核。
- TransitionPolicy：5 级状态机只输出 keep/promote/demote/review/suppress 建议，不直接改 store。

3. MemoryStoreBridge
- 采用 JSONL append-only 事件流。
- add_candidate 追加候选记录。
- mark_tombstone 只追加 tombstone 事件，不物理删除。
- suppress_active_recall 只追加主动召回抑制事件。
- record_use_feedback 把记忆被 Planner 使用后的成功/失败反馈回写为事件。
- export_snapshot 只导出摘要级快照。

4. MemoryWriteFilter / EvidenceGate
- 没有 evidence_refs 不允许直接 append。
- confidence_score < 0.45 进入 review。
- privacy_risk_score >= 0.80 阻断进入普通 PlannerContext。
- pollution_risk_score >= 0.60 进入 review。
- conflict_score >= 0.70 进入 review。
- L5 只允许系统规则、工程宪法、不可变审计指针、长期稳定能力规则，不保护用户隐私/污染/错误记忆。

5. ForgetReviewRouter
- 用户显式忘记请求优先级最高。
- 显式忘记请求会生成 suppress_active_recall + tombstone + delete_review。
- direct_delete_allowed 永远 False。
- L5 冲突进入 retention_exception_review，不覆盖用户显式忘记请求。

6. MemoryRecallRouter
- 只输出 PlannerMemoryHint。
- 只含 sanitized_summary、digest、evidence_refs、recall_score。
- 不输出原始正文。
- tombstone、active_recall_suppressed、高隐私、低置信内容不得主动召回。
