天工造物 v2.0 - 临渊者 L6.72.5 任意位置启动热修报告
============================================================

一、问题归因

用户截图中的报错不是前端渲染层崩溃，而是 Windows 启动器路径容错不足：
- 旧 BAT 依赖固定 ..\\.. 路径推导项目根目录。
- 当用户从 01_启动入口/Windows、压缩包临时视图、复制入口、或非预期工作目录启动时，可能找不到 00_ASCII_START_HERE/python/START_DESKTOP_L6710.py 或 backend/frontend/desktop。
- 随后 Python 启动链进入缺路径状态，CMD 显示 The system cannot find the path specified，并返回 exit code 3。

二、本轮修复

1. Windows BAT 任意位置根目录发现
- 改造 00_ASCII_START_HERE/windows/*.bat。
- 改造 01_启动入口/Windows/*.bat。
- 新增根目录 START_FROM_ANYWHERE_AUTO_L6725.bat。
- 新增根目录 SELF_CHECK_FROM_ANYWHERE_L6725.bat。
- 支持从脚本目录、当前目录、上级目录、Desktop、Downloads、Documents 自动搜索项目根。
- 找不到根目录时输出明确诊断：必须完整解压 ZIP，不能在压缩包视图里直接双击 BAT。

2. 诊断入口降级为纯 Python
- SELF_CHECK_L6719.bat 不再要求 tkinter 可用才启动。
- DEPENDENCY_CHECK.bat 保持纯 Python 诊断。
- 这样缺 Tk 的 Windows 机器也能看到依赖诊断，而不是入口直接退出。

3. Python 入口根目录发现增强
- 00_ASCII_START_HERE/python/START_DESKTOP_L6710.py 支持 LINYUANZHE_ROOT_HINT / TIANGONG_ROOT_HINT / cwd / __file__ 多源定位。
- SELF_CHECK_L6710.py 同步增强。
- DATAUP_SAFE_UPDATE_L6717.py 同步增强。
- 01_启动入口/通用Python 同步更新。

4. macOS/Linux 入口同步容错
- 00_ASCII_START_HERE/linux_macos/*.sh 已改为向上查找 + 常用目录搜索。
- 01_启动入口/Linux/*.sh 已同步。
- 01_启动入口/macOS/*.command 已同步。

5. 版本标记
- frontend/linyuanzhe_frontend/version_info.py 提升到 L6.72.5。
- README 与 VERSION_PRODUCT 已同步说明。
