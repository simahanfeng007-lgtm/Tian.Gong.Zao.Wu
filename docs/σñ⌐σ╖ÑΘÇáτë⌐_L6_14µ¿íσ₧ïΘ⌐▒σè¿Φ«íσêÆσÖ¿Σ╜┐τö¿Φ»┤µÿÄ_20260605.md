# 天工造物 L6.14 模型驱动计划器使用说明

## 定位

L6.14 在 L6.13 的 Runtime 审计链基础上新增“自然语言任务 → 结构化计划 → 受治理执行”闭环。

模型只允许输出 JSON 计划，不允许直接执行工具；所有计划步骤必须经过：

```text
ModelPlanner → PlanValidator → RiskClassifier → PermitGateway → RuntimeToolRegistry → Adapter → AuditBridge
```

## 计划器模式

新增参数：

```bash
--planner-mode rule_only
--planner-mode model_suggest
--planner-mode model_required
```

含义：

- `rule_only`：默认值，只识别显式 DSL，例如 `write a.txt :: hi && read a.txt`。
- `model_suggest`：显式 DSL 未命中时，调用模型生成结构化计划；计划失败则回退普通对话。
- `model_required`：显式 DSL 未命中时，必须由模型生成计划；计划失败不回退普通对话。

## 预览计划

交互模式下：

```text
/plan 请检查项目并打包
```

该命令只展示结构化计划，不执行真实工具。

## 执行自然语言计划

```bash
python run_agent.py --mock --tool-mode runtime_governed --planner-mode model_suggest --workspace ./workspace --once "请检查项目并打包"
```

Mock 模式下会生成确定性计划，适合离线验证。

## 安全边界

模型计划器只允许生成以下工具：

- `list_dir`
- `read_file`
- `write_workspace_file`
- `run_python_quality_check`
- `create_zip_package`

禁止：

- 任意 shell 命令；
- 未注册工具；
- 绝对路径；
- `../` 越界路径；
- 敏感凭据路径；
- `model_chat` 递归模型调用；
- 插件或模型绕过 Runtime 直接执行工具。

## 当前限制

- 真实模型计划质量依赖 Provider 输出是否严格遵守 JSON。
- 第一版不做复杂代码修复规划，只做受控计划生成。
- 长期记忆写入、桌面控制、任意终端执行仍未开放。
