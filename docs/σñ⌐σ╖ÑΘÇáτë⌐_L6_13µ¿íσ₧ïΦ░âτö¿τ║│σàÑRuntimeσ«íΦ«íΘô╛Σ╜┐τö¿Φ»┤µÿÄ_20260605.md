# 天工造物 L6.13 模型调用纳入 Runtime 审计链使用说明

## 定位

L6.13 不是前端阶段，也不是安装包阶段。它只解决一件事：把普通聊天中的模型调用，从 `tiangong_agent_shell` 直接调用模型客户端，升级为通过 `tiangong_agent_runtime` 的受治理链路执行。

链路为：

```text
Shell 会话
→ RuntimeEntry.run_model_chat
→ ToolInvocation(model_chat)
→ RiskClassifier(A2)
→ PermitGateway(auto allowed)
→ RuntimeToolRegistry
→ model_chat_adapter
→ AuditBridge
→ PublicProjectionBridge / RuntimeReport
```

## 使用命令

Mock 模型仍然可用：

```powershell
python run_agent.py --mock --once "你好"
```

交互模式：

```powershell
python run_agent.py --mock
```

工程工具链仍然可用：

```powershell
python run_agent.py --mock --tool-mode runtime_governed --workspace ./workspace --once "write demo.txt :: hello && read demo.txt"
```

## L6.13 的核心变化

1. 普通模型聊天会生成 `model_chat` 审计事件。
2. `model_chat` 风险等级为 A2，默认自动放行。
3. `tool_execution_mode=disabled` 不再阻断普通聊天；模型调用单独通过 Runtime 审计链执行。
4. 工具执行仍按原策略：A1-A3 自动、A4 确认、A5 阻断。
5. 审计、报告、公共投影不会导出完整 prompt、完整 messages 或 API Key。

## 安全边界

- `tiangong_kernel/` 主体未改动。
- 模型调用不获得工具执行权。
- `model_chat_adapter` 只调用模型客户端，不执行文件、终端、桌面或网络工具。
- messages、model_config、model_client 只放在 `TurnContext` 内部上下文，不放进 `ToolInvocation.arguments`。
- 审计只记录 provider、model、messages_count、last_user_preview、stream、api_key_configured 等摘要字段。
- API Key 输出继续脱敏。

## 当前限制

- 真实 API 联调仍需要用户提供真实 Base URL / API Key / model。
- 还没有专用 DeepSeek / Qwen / GLM / MiniMax / MiMo provider 细化调用器；当前仍优先走 OpenAI-compatible 最小协议。
- 没有桌面端。
- 没有一键安装包。
- 没有三端安装验收。
- 没有开放任意终端、桌面控制或长期记忆写入。

## 验收重点

```powershell
python run_agent.py --mock --once "你好"
python run_agent.py --mock --once "rm 是什么"
python run_agent.py --mock --tool-mode runtime_governed --workspace ./workspace --once "write a.txt :: hello && read a.txt"
python -m compileall -q tiangong_agent_shell tiangong_agent_runtime run_agent.py
pytest tests/test_l6_13_model_chat_runtime_audit.py tests/test_l6_13_no_kernel_pollution.py -q
pytest -q
```

通过后，L6.13 状态为：模型聊天、工具执行、确认票据、审计回放、运行报告都已经进入同一条 Runtime 证据链。
