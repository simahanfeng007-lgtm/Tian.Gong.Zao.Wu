三、L4 禁止实现

1. SkillRegistry、ToolRegistry、ToolGroupRegistry。
2. ModelClient、FileSystemClient、NetworkClient、ShellRunner、DesktopController。
3. MemorySystem、RetrievalSystem、LearningSystem、AffectiveSystem。
4. RecoverySystem、ReplaySystem、PluginHost、ConnectorPlatform、UI 前端。

四、结论

L4→L6 交接口径已冻结。L4 不实现 L6。

五、记忆与遗忘 sink 专项补丁说明

新增代码 surface：
1. memory_sink。
2. forgetting_sink。
3. retrieval_sink。
4. learning_sink。
5. affective_feedback_sink。
6. privacy_retention_sink。

新增 requirement：
1. L4ToL6MemorySinkRequirement。
2. L4ToL6ForgettingSinkRequirement。

链路口径：
动作结果 / 观察结果 / 用户反馈
→ L4 标准化结果与 ActionResultCognitiveSinkHint
→ L5 审查记忆写入、隐私、保留、删除、审计和 Tombstone 边界
→ L6 记忆/遗忘插件执行真实存储、召回、巩固、修订、遗忘或删除
→ 返回引用与审计摘要
→ L3/L2 记录状态建议与状态事实。

边界：
L4 不写记忆，不执行遗忘，不实现 MemorySystem / ForgettingSystem / RetrievalSystem / LearningSystem。

六、自我学习 / 自我迭代 / 自我进化专项补丁说明

新增 L6 requirement：
1. L4ToL6SelfLearningSinkRequirement。
2. L4ToL6EvolutionValidationRequirement。
3. L4ToL6EvolutionCommitRequirement。
4. L4ToL6PostCommitObservationRequirement。

链路口径：
学习信号 / 候选 / 验证结果 / 人工复核
→ L3 CandidateValidatedToCommitAdvice 与 EvolutionCommitFlowAdvice
→ L5 边界许可、人工确认、租约、审计和回滚锚点
→ L4 requirement-only 承载
→ L6 后续真实学习、验证、提交、观察或回滚服务
→ 返回引用与审计摘要
→ L3/L2 记录建议与状态事实。

边界：
L4 不生成 patch，不应用 patch，不提交，不热切换，不回滚，不实现真实学习或进化系统。

七、自愈系统专项补丁说明

1. L4SelfHealingHandoffRef。
2. L4FailureRecoveryRequirementBundle。
3. L4PostRecoveryValidationRequirement。

链路口径：
ActionFailureReturnEnvelope / failure_ref
→ L1 FailureDiagnosisRequest
→ L2 FailureDiagnosisState / RecoveryPlanState
→ L3 SelfHealingFlowEnvelope

八、Context / BeliefState / WorldState 安全边界专项补丁说明

新增 L4 投影：
1. L4ContextSafetyProjection。
2. L4ToolOutputContextProjection。
3. L4ModelOutputContextProjection。
4. L4ObservationBeliefWorldProjection。

链路口径：
ToolResultEnvelope / ModelActionResult / ObservationReturnEnvelope
→ L4 context safety projection
→ L5 权限、隐私、信任、taint 和指令资格边界审查
→ L6 future context assembler / belief updater / world state reconciler

边界：

L4 不装配上下文、不更新 BeliefState、不同步 WorldState。
