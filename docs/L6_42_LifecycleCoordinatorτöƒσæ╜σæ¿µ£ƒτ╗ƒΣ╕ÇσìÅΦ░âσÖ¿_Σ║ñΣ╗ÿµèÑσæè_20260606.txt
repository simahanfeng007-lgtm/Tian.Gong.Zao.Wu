1. tiangong_agent_runtime/lifecycle_coordinator.py
生命周期统一协调器，输出 LifecycleRouteBundle / LifecyclePlannerHint / LifecycleStatusRoute。

2. tiangong_agent_runtime/self_healing_execution_route.py
SelfHealing 接入路由。只生成自愈诊断、恢复、验证、回归候选。

5. tiangong_agent_runtime/self_iteration_route.py
SelfIteration 接入路由。只生成改进候选、版本槽候选、验证需求、回滚需求，不自动 apply / merge / hot switch。

6. tests/test_l6_42_lifecycle_coordinator.py
L6.42 生命周期统一协调器与四条生命周期路径边界测试。

四、关键边界

1. LifecycleCoordinator 只做候选协调，不做执行。
- no_second_runtime=True
- no_direct_execution=True
- no_tool_invocation=True
- no_model_dispatch=True
- no_budget_mutation=True
- no_memory_write=True
- no_skill_registry_write=True
- no_tool_registration=True
- no_patch_apply=True
- no_hot_switch=True
- no_kernel_mutation=True

2. SelfHealing 不能自动修复。
禁止：
- invokes_tool
- applies_patch
- writes_file
- performs_rollback
- performs_hot_switch
- mutates_kernel

3. AutonomousLearning 不能自动学习写库。
禁止：
- writes_knowledge
- writes_skill_registry
- activates_skill
- produces_tool
- registers_tool
- invokes_tool
- dispatches_model
- mutates_kernel

4. FreeWill 不能抢当前用户任务。
新增规则：
- active_user_task=True 且 user_allowed_autonomy=False 时，FreeWill 降级为 FW0 观察。
- FW3/FW4 必须 requires_ticket=True。
- FW5 不允许生成 FreeWillCandidateRoute，必须在路由前硬拒绝。
- AutonomyLease 不授予执行权，不授予预算修改权，不允许绕过用户任务。

5. SelfIteration 不能自动合入或热切换。
禁止：
- applies_patch
- merges_change
- performs_hot_switch
- activates_version_slot
- writes_file
- invokes_tool
- mutates_kernel

五、生命周期优先级冻结

L6.42 使用如下优先级：

1. P0_safety_data_credential_irreversible_risk
2. P1_current_user_task_closure
3. P1_current_task_failure_recovery
4. P2_user_requested_learning
5. P2_user_confirmed_iteration
6. P3_self_healing_candidate
7. P3_learning_after_delivery
8. P4_iteration_after_stability
9. P5_free_will_exploration

核心原则：
- 当前用户任务闭环优先。
- 当前任务失败恢复优先于学习和迭代。
- 用户显式学习/迭代优先于系统自主候选。
- 自由意志探索最低优先级。
- P0 安全/凭证/不可逆风险最高。
