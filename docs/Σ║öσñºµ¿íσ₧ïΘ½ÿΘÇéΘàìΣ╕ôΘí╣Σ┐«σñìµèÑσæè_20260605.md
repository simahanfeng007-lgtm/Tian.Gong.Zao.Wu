# 五大模型高适配专项 hotfix3 修复报告

## 修复依据

依据 `天工造物_L4补充专项_五大模型高适配质检报告_20260605.txt` 的 P1/P2 结论执行最小补丁：修正官方文档引用、补 L1 显式端口、补 L4 public export、修正 L5 影响评估口径、修复新增中文文档文件名 mojibake，并按用户追加要求处理 MiMo 模型 ID 小写与 plan/普通 API 双面支持。

## 关键修复点

1. DeepSeek V4 factsheet 官方引用从 V3.2 页面修正为 DeepSeek V4 Preview 官方页面 `news260424`。
2. MiniMax M3 factsheet 官方引用统一到 `platform.minimax.io` 官方模型、AI Coding Tools 与 Token Plan 文档。
3. MiMo 模型 ID 全部归一化为小写：`mimo-v2.5-pro`、`mimo-v2.5`、`mimo-v2.5-asr`、`mimo-v2-flash`、`mimo-v2-pro`、`mimo-v2-omni`。
4. MiMo 同时支持 `token_plan_api` 与 `ordinary_api` 两个 API 面；二者都只以 endpoint/ref 与 credential scope/ref 表达，不写死 endpoint，不允许插件选择或直连。
5. L1 新增 `ModelProviderPort` 与 `ModelInvocationPort`，均为 provider-neutral、ref-only，不调用模型、不裁决 provider。
6. L4 新增 `MiMoApiSurfaceDescriptor`、`mimo_api_surface_descriptors()`、`MiMoTokenPlanRequestMapper`、`MiMoOrdinaryApiRequestMapper`。
7. L4 `__init__.py` 补充五大模型 provider adapter 公共导出。
8. L5 影响评估改为基于“情志 hotfix2 → 五大模型 hotfix3”的明确比较口径。
9. 新增中文交付文档文件名已从 mojibake 修复为 UTF-8 可读名称。

## 是否建议继续 L6

建议继续 L6 策划/一般插件开发；不建议直接进入 L6 真实模型插件开发。L6 插件只能声明 ModelCapabilityRequirement，不得裸调模型 SDK；真实模型调用必须经 L3 编排、L5 许可、L4 provider adapter。
